## JavaScriptとは
HTMLとCSSで作られたWebページは**静的コンテンツ**と呼ばれます。これはHTMLでは事前に決められた内容(HTMLファイルに書いた内容)を表示する機能しかなく、Webページを閲覧している人が行ったアクションに応じてWebページの表示内容を変えることはできないからです。

これに対して、JavaScriptを用いたWebページは**動的コンテンツ**と呼ばれます。JavaScriptを使用することで入力項目を非表示にしたり再表示したり、Webページ上で動画再生をすることが出来るようになったりアニメーションをつけることができたりと複雑なことが出来ます。

本来、JavaScriptはブラウザ上でしか動作しないプログラムでしたが、Node.jsがリリースされたことで、Node.jsでJavaScriptを実行することが出来るようになりました。

 JavaScriptのJavaスクリプトの歴史などネットで調べてみると面白いです。

## JavaScriptの勉強を始める前に
JavaScriptの書き方を勉強しようとすると、専門用語が多く理解が難しいと思います。  
そこで、事前に必要な言葉をここで覚えておきましょう。

### プログラムにおける「データ」についてと「オブジェクト指向」
#### データとは？
**データ**と聞いて何を思い浮かべるでしょうか？「自分のデータを教えてください」と言われたら何を答えますか？名前、年齢、性別、居住所、家族構成などでしょうか。そう、これがデータです。  
「自分のデータを答える」プログラムを作るとしたらどうなるでしょうか。アルゴリズムを書くとして、たとえば年齢が33歳だったら「私の年齢は33歳です。」という情報をどうやって書けばいいでしょうか？  
普通に文字列で書いてもいいですが、もしプログラムを実行する人が外国人だったら、外国語でも書いておく必要があります。そもそも文章で返す必要はなく、「年齢 は 33」という意図が伝わればいいと考えると、もっとスマートに書ける気がします。  
そこで世界共通といってもいい算数の数式を真似して、「年齢 = 33」という書き方で伝えよう、という考えが出てきました。共通言語になっていると言っても過言ではない英語を用いで「age = 33」とするとより伝わりやすいですね。このように、プログラミングにおけるデータとは、「〇〇 = xx」という形式で、「何(〇〇)」が「どう(xx)」と表します。  

#### データ構造
自分のデータを出力する、という想定でいくと、先ほどの「age = 33」というデータは誰のものか、というのも表す必要があります。誰、という情報は「name = 〇〇」で表せそうです。ageとnameという2つのデータがありますが、これは2つとも同じ存在 = 自分のことです。日本語で表すと「私は〇〇という名前で、33歳です。」となります。「私は」という部分をプログラムに伝える必要がありますが、「私 = 名前 = 〇〇」だとよくわかりませんね。  
こういった状況に対して、昔の頭のいい人は下のように書くと分かりやすいかも、と考えました。
```
watashi {
    name = 〇〇
    age = 33
}
```
「私」に含まれるデータを中括弧`{}`で囲んであげる形ですね。  
このように、データというのは入れ子構造(一つのデータの中に複数のデータが含まれる上下関係があること)になっていることが多いです。これを業界用語で**データ構造**と呼びます。データを扱うプログラミングをする場合、まずはこのデータ構造を考えます。  
住所や家族構成も含めて自分のデータを構造化してみると、
```
jibun { 
    name = 〇〇
    age = 33
    address = xxxxxxxx
    sex = female
    family = {
        father = 〇〇
        brother = 〇〇
    }
}
```
みたいになりますね。これがデータ構造です。そして、`jibun`というデータの塊のことを**オブジェクト**と呼びます。  
オブジェクト指向プログラミングとは、扱うデータを全てオブジェクトとして表した上で、このオブジェクトを操作するアルゴリズムを記述することです。  

#### オブジェクトのもう一つの機能、「関数」
オブジェクトについて、もう一つ大事な機能があります。先ほどの例ですと、自分のデータを構造化しましたが、「自分のデータを答える」プログラムを作ろうと考えると、「答える」という部分がまだありません。「答える」というのはデータではなさそうですが、jibunというオブジェクトに「教えて」欲しいですよね。データを持ってるオブジェクトなので。  
そこで、このjibunというオブジェクトに「答える」という機能を追加してみましょう。下記のように書いてみました。
```
jibun { 
    name = 〇〇
    age = 33
    address = xxxxxxxx
    sex = female
    family = {
        father = 〇〇
        brother = 〇〇
    }

    asking = function{
        "私の名前は" + name + "で、年齢は" + age + "です。"
    }
}
```
簡単にするために名前と年齢を日本語で答える、みたいにしてます。このように、データではなくオブジェクトに「何か」をしてもらいたい時、その「何か」を算数の言葉になぞらえて**関数(function)**として記述することにしました。人によっては関数のことをメソッド(Method)と呼ぶ人もいます。同じ意味です。  

#### オブジェクト指向プログラミング
ここから、実際のプログラミング言語で決められている機能の話をします。  
##### 変数と定数、関数
オブジェクトとは、データと関数を構造的に記述したものです。JavaScriptではデータのことをプロパティ(Property)と呼びます。関数はfunctionです。プロパティも関数も、「name = 〇〇」のように、「変数(Variable) = 値(Value)」の形で記載します。変数のことをKeyと呼んだりもします(キーワードのことですね)。  
自分の名前は結婚とかで変わる可能性がありますし、年齢は1年毎に増えていくので「変数」というのも分かりますが、性別は滅多なことでは変わらないですよね。その場合、ほとんどのプログラミング言語では**定数**という呼び方で他のプログラムで変更されないようにすることもできます。この「変数」と「定数」については、実際にプログラムを書いていきながら使い方を学んでいってもらえればと思います。

##### クラス
今までは、自分のデータを答える、という例で話してきました。これをもう少し汎用的にしてみましょう。新しいプログラムを作るとして、今までと同じように名前や年齢を答える人の数を増やしたい、となったとします。自分とはデータの「値(Value)」は違うけど、データ構造は同じオブジェクトが複数必要になった場合です。  
このような場合に、**クラス(Class)**という機能が非常に便利に使えます。クラスとは、オブジェクトの設計図です。データ構造だけを書いておいて、実際のプログラムで使いたい時にクラスという設計図を元に値をセットしてオブジェクトとして扱う、そんな機能があります。今回の例をクラスを用いて書いてみると、下のようになります。

```
class Person {
    name
    age
    address
    sex
    family

    constructor = function(name,age,address,sex,family) {
        // 引数の値を変数にセットする
    }

    asking = function{
        "私の名前は" + name + "で、年齢は" + age + "です。"
    }
}

watashi = new Person("〇〇",33,"xxxxxxx",female,{father = "〇〇" brother = "〇〇"})

son = new Person("△△",4,"xxxxxxx",male,{father = "〇〇" mother = "〇〇" sister = "〇〇"})

watashi.asking()
son.asking()
```
かなりプログラムっぽくなりましたね。最初に```class Person```と書いてありますが、これは今まで`jibun`として書いていたオブジェクトを、クラスとして書きますよ、という宣言です。Personは「人」という意味ですね。  
`Person`の中身は、`jibun`の時と同じプロパティが書いてありますが、値がありません。そして、`constructor`という関数が追加されています。これは**コンストラクタ**という特別な関数で、クラスを書くときは必ず書く必要があります。これは、クラスからオブジェクトを作る時にプロパティに値をセットするための関数です。  
最後の4行でこのクラスを使ってオブジェクトを作ってasking関数を実行する、という命令を書いています。`watashi`という変数の値として`new Person()`という関数が書いてあります。これがクラスからオブジェクトを作る方法です。`new`というキーワードがオブジェクトを作ってくれます。後ろの`Person(~)`はクラス名(~)という書き方ですが、実はコンストラクタを実行しています。  
この`watashi = new Person(~)`を**オブジェクトの生成**と呼び、生成されたオブジェクトを**インスタンス(instance)**と呼びます。インスタンスという言葉は、英語で「実態」とか「実例」とかの意味です。クラスという設計図に値を与えて実体化する、これをオブジェクト生成、インスタンス化と呼んだりします。

オブジェクト指向プログラミングとしては、このクラスを用いて様々なことをできるように機能がたくさんあったりしますが、全部を説明するのは長いので、細かいところはプログラミング学習で学んでいってください。
